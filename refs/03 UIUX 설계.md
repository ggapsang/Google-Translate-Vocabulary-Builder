## 디자인 철학

### 핵심 원칙
1. **심플함**: 학습에 방해가 되지 않는 깔끔한 디자인
2. **직관성**: 설명 없이도 사용 가능한 인터페이스
3. **일관성**: 구글 번역 UI와 자연스럽게 어울리는 디자인
4. **접근성**: 색각 이상자, 키보드 사용자 고려

### 디자인 시스템

- **컬러**: Google Material Design 팔레트 기반
- **타이포그래피**: system-ui, -apple-system, 'Noto Sans KR'
- **아이콘**: Material Icons
- **간격**: 4px 그리드 시스템

│  Google 번역                        │

│  영어    →    한국어                 │

│                 │  invoice      🔊  │
│  Of course,     │  ┌─────────────┐  │
│  most indoor... │  │ 정의  번역   │  │

#### 버튼 디자인

```css

  color: white;
  padding: 8px 16px;
  border-radius: 4px;

  background: #34a853;
  color: white;
  cursor: default;
}
```

```

│  저장 완료              │
│  "invoke" + 예문        │
└────────────────────────┘
```
- 2초 후 자동 사라짐
- 오른쪽 하단 토스트 메시지

- **복습 완료**: masteryLevel +1, 다음 복습일 연장
- **잘 모르겠음**: masteryLevel -1, 다음 복습일 단축

## 접근성 고려사항

### 키보드 네비게이션
- Tab: 포커스 이동
- Enter: 선택/실행
- Esc: 모달 닫기
- 방향키: 목록 탐색

### 스크린 리더
- ARIA 레이블 적용
- 의미있는 alt 텍스트
- 포커스 인디케이터

### 색각 이상
- 색상에만 의존하지 않는 정보 전달
- 아이콘 + 텍스트 병행
- 충분한 명도 대비 (WCAG AA)
