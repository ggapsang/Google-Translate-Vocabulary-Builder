## 참조 
 [[06 디렉토리 구조]] 
 
 - `manifest.json`에서 권한과 엔트리(콘텐츠/백그라운드/사이드 패널/팝업) 정의
 - `content/`의 스크립트가 번역 페이지 DOM을 관찰해 
 - [[02 기술 스펙]]에 나온 파싱 전략으로 단어·품사·예문을 캡처하고, 저장 
 - [[03 UIUX 설계]]의 디자인 원칙(심플함, 직관성, 구글 번역과의 일관성) 유지
 - 감지된 데이터는 `lib/`의 저장소/Export/복습 로직을 거쳐 `chrome.storage.local`이나 CSV/Sheets로 내보낼 수 있음
 - Phase 2/3 기능(Spaced Repetition, Sheets 연동, 알림)은 `background/service-worker.js`에서 향후 확장

- UI는 팝업·사이드패널·테스트/통계/설정 탭으로 구분
	- **사이드패널:** 단어 카드 목록/필터/검색/상세 보기·복습 버튼을 제공
	- **테스트 탭**: 객관식·빈칸·스펠링 등의 모드
	- **통계 탭**: 학습 현황·주간 활동·언어·품사 분포
	- **설정 탭**: 테마·언어·목표·알림·데이터 관리 기능
	- **팝업**: 빠른 현황·복습 목록·바로가기 버튼을 제시
	- 모든 UI는 [[01 프로젝트 기획안]]의 “학습 도구 + 독립 UI” 가치를 구현
	- 각 문서는 [[05 데이터 구조]]가 보여주는 데이터 스키마와 연계되어 저장·내보내기·복습 판단을 일관되게 수행
	- 문서와 코드가 계속 맞춰지도록 이 문서에서 유지 방향을 제시함
